Informações sobre JWT

Token de Acesso

  • • Usado para autenticar requisições à API
  • • Tempo de expiração limitado (segurança)
  • • Deve ser incluído no header Authorization
  • • Formato: Bearer <token>

Refresh Token

  • • Usado para gerar novos tokens de acesso
  • • Tempo de expiração mais longo
  • • Deve ser armazenado com segurança
  • • Evita necessidade de login frequente
Importante

Mantenha seus tokens seguros e nunca os compartilhe. Use sempre HTTPS em produção e considere implementar rotação automática de tokens.
